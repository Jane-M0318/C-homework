三、	功能设计记录
3.1 整体设计
3.1.1 系统功能描述
（1）n位选手报名参赛。选手报名时，输入选手姓名、编号和参赛歌曲。参赛歌曲从备选曲目中筛选（备选曲目保存在文件songs.txt中）。
（2）m（m 3）位评委为选手打分，评分最高10分，最低0分。
（3）计分规则：每个选手的m个得分中，去掉一个最高分和一个最低分，再取平均分作为该选手的最后得分。
（4）根据n个选手的最后得分，从高到低输出选手的得分名次榜，确定冠军名单。
（5）根据各位选手的最后得分与各位评委给该选手所评分数的差距，对每个评委评分的准确性和水准给出一个定量的评价，从高到低输出各位评委得分的名次榜。
（6）颁奖：从文件（youwin.txt）中读取“获胜字符画（YOU WIN）”，显示在控制台中，同时显示冠军选手姓名信息。

3.1.2 程序整体框架
基本框架（头文件---结构体定义---自定义函数声明---main函数---自定义函数实现）；
（1）	根据题目描述，为参赛选手、评委定义结构体；
（2）	根据功能要求，提炼自定义函数，分析函数原型，描述函数功能；
（3）	定义menu函数，显示用户可以选择的功能选项，并接收用户的选择；
（4）	在main函数中，循环执行menu，根据用户的选择，通过switch/case结构，执行对应case下面的功能函数；

3.1.3 结构体设计
（1）	创建参赛选手结构体，确定结构体需要包含的数据成员、确定每个数据的类型。
成员可以包括：选手编号、选手姓名、参赛曲目、评委组打分、选手最终得分。
		选手编号：int
		选手姓名：字符串
		参赛曲目：字符串
		评委组打分：数组
		选手最终得分：float
（2）	创建评委结构体
成员可以包括：评委的ID、评委打分水平等级
			评委的ID：int
评委打分水准等级分值：float

小结：在设计结构体时，因为裁判和选手人数不定的原因，纠结了很久定义数组个数时应该使用动态分配，还是直接把M 在头文件里define得特别大。最后因为人数比较可控，在一定范围内并且没必要释放空间，于是选择了比较简单的后者做法。

3.1.4 main函数设计
在main函数的前面放上自定义函数的声明，在main函数的后面放上这些函数的实现
这样相关的函数，就可以在main函数中加以调用了，可先来设计main函数中的代码。

设计思路：
（1）根据功能要求，main函数中需要定义相关局部变量：
选手的默认人数
评委的默认人数
选手数组
评委数组
备选曲目数组
其它变量
（2）main函数中需要调用相关自定义函数来执行特定功能，可以用到switch/case结构。在switch中，调用功能菜单函数，根据功能菜单函数返回的序号，执行对应的case，运行相应的功能；
（3）在每一个case中，分别调用对应的自定义函数执行功能；
（4）在菜单函数中会有一个退出选项（0选项，执行exit()函数退出程序），因此main函数中我们可以放置一个无限循环语句while(1) {}，来让程序一直运行，直到用户选择退出。

6、评委给某位选手打分的函数 
功能要求：M位裁判为某位选手打分
函数原型：
本函数实现M位裁判为某位选手打分。
本函数在“计算所有选手最终得分的函数”中被循环调用。
函数形参：选手数组、评委数组、某位选手的编号、评委数目。
选手数组里面保存了参赛的选手，数组里面的元素的数据类型是我们定义的选手结构体，结构体中的scores[j]数组用来保存第j位评委为选手i所打的评分。
结构体里面的成员，通过成员运算符“.”，来获取相应的成员，如p[i].scores[j]，通过成员运算符“.”获取了第i位选手的第j位评委的评分。
读入选手的编号，读入评委的评分，分别保存在结构体对应的数据成员中。


7、计算所有选手最终得分的函数
功能要求：在获取所有选手评分的基础上，计算所有选手的最终得分。
(1) 外层循环控制参赛选手的编号i从1变化到n，当第i个选手上场时，调用“评委给某位选手打分的函数”函数，完成对 第i个选手的打分；
(2) 内层循环控制给选手评分的评委的编号j从1变化到m，以此得到第j个评委给第i个选手的评分，并将其累加到第i个选手的总分中，同时求出评委给第i个选手评分的最高分max和最低分min；
(3) 第i个选手的m个得分全部输入并累加完毕后，去掉一个最高分max和一个最低分min，然后取平均分作为第i个选手的最后得分，显示该选手的得分；
(4) 完成n个参赛选手的最后得分的计算。
用户在菜单函数中选择功能4，执行打分和计算最终得分的功能；

9、评委评分水准等级分值的计算函数
功能要求：计算评委评分水准等级分值。
设计思路：
计算评委给选手的评分与选手最终得分之间的误差，来评估该评委的水平。假设第j个评委给第i个选手的评分为f[i][j]，第i位选手的最终得分为sf[i]：
利用计算公式对第j个评委的评分水平pf[j]进行定量评价。pf[j]值越高，说明评委的评分水平越高。
该分值将用于对评委的排名。
本函数在“评委排名函数”之前运行，以获取评委的评分水准等级分值。可以在对应case中调用执行。

四、	课程设计小结（设计心得、对改进系统功能的思考、对C的理解等）
一、设计心得
1. 模块化设计的重要性
将系统划分为多个功能模块（菜单、选手管理、评委管理、评分计算、排名显示等），使代码结构清晰，便于调试和维护。每个函数负责单一功能，提高了代码的可读性和可重用性。
2. 数据结构的合理选择
•	使用结构体存储选手和评委信息，符合现实世界的对象模型
•	数组作为主要的数据存储容器，简单高效
•	二维数组处理评委打分表，直观易懂

二、对C语言的理解加深
1. 文件操作的熟练度
•	文本文件的读写（fopen、fclose、fgets、fscanf）
•	数据格式的处理（如去除换行符）
3. 算法实现能力
•	排序算法（冒泡排序）的实际应用
•	复杂计算公式的实现
